Downloads

Playdar is under heavy development, so to keep up-to-date you're advised to build from source. Binary packages and installers are provided periodically.

Building from source

git clone git://github.com/RJ/playdar.git
cd playdar/playdar-daemon
less INSTALL.txt
cd build
cmake ..
make

If you don't have git, or you don't have all of playdar's deps (check INSTALL.txt), read the following instructions as well:

Linux

eg. Ubuntu: 
apt-get install \
    sqlite3 libsqlite3-dev \
    cmake \
    libtag1-dev \
    liburiparser1 liburiparser-dev \
    libossp-uuid-dev libossp-uuid15

Mac

You need MacPorts:

    port selfupdate
    port update installed
    port install cmake boost ossp-uuid uriparser taglib git-core

Windows

We've been building on Windows with Visual Studio 8. Other versions will probably work too.
You'll need to find the deps yourself (suggest getting boost from boostpro.com).
A project file is in the repo.
Ask in our IRC channel if you need help.

Running Playdar

Check the INSTALL.txt, make sure you run from the daemon dir so ./plugins and ./www exist. Create the database the first time by doing "sqlite3 ./collection.db < etc/schema.sql"

Binaries

These will be outdated. Visit the irc channel (Freenode, #playdar), and check the /topic or ask if there is a newer one. Another option is to check the mailing list or mailing list archives.

Playdar Installer (NSIS installer by lozzd)

NB: windows installer comes with the normal daemon, and a wrapper app:

  • Wraps playdar.exe and scanner.exe completely, and shows log output.
  • Minimises to tray
  • Colour coding of important events in log
  • Provides Balloon notifications for events
If you don't need to use the wrapper, the installer puts the playdar daemon in the install directory too.
